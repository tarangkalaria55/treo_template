Message

treo-message is a customizable component for displaying all kinds of messages across your app. They can be controlled from your components as well as from the TreoMessageService.

Exported as: treoMessage

Module

Properties

Name Description Default
@Input()
name: string
 A unique name for the message box to access it from TreoMessageService. undefined
@Input()
appearance: TreoMessageAppearance
 Appearance of the message box. 'soft'
@Input()
dismissed: null | boolean
 Whether the message box dismissed. null will disable the dismiss feature. null
@Input()
showIcon: boolean
 Whether the icon is shown. true
@Input()
type: TreoMessageType
 The type of the message box. 'primary'
@Output()
afterDismissed: true
 An event emitted after the message box dismissed. -
@Output()
afterShown: true
 An event emitted after the message box shown. -

Icons

The info, success, warning and error type message boxes have default icons and they are visible by default. If you wish to hide those icons, use the [showIcon]="false" input.

Example
Success message with no icon

Custom icons

All message boxes come with custom icon support. To use a custom icon, simply use treoMessageIcon attribute on an element you wish to use as the icon. If there is a default icon for the message box, custom icon will override it.

Example
Info message with a custom icon

Dismissible messages

Dismissible messages can be removed from the view by clicking their dismiss (x) button. In order to get the dismiss button visible, the [dismissed] input must be explicitly set either to true or false. Setting the [dismissed] input to null or simply removing it from the element will remove the dismiss button.
