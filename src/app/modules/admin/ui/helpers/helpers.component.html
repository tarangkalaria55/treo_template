Form field helpers

By default, Treo changes the default form field appearance to fill and heavily modifies it to provide a more unique and consistent look. We recommend NOT using any other form field styles as they are not optimized for Treo.

In addition to the heavy modifications, Treo also provides set of helper classes to further modify the style of the form fields.

Required classes

  • .treo-mat-textarea

    This helper class is required if you are using a textarea as your input. Since Angular Material doesn't differentiate a textarea from a normal input while using them with mat-form-field, this helper class is required for textarea inputs to look good and consistent.

Optional classes

  • .treo-mat-dense

    Provides set of adjustments to make the fields denser. Dense fields are suitable for non-form usages and places where there isn't much space to work with such as small dialogs, menus, popovers and etc.

  • .treo-mat-rounded

    Provides set of adjustments to make the fields look fully rounded.

  • .treo-mat-no-subscript

    Provides set of adjustments to remove the bottom spacing where hint and error messages placed. This helper is particularly useful to align the field easily if you don't need to show error messages or hint text. For example a search bar in the toolbar uses this helper to fit the field to the toolbar without having an extra bottom spacing.

  • .treo-mat-emphasized-affix

    Provides set of adjustments to emphasize the field prefix and suffix by changing their background color and adding a border around them.

Usage

<mat-form-field class="{{getFormFieldHelpersAsString()}}">...<mat-form-field>

TailwindCSS based helpers

As mentioned before, Treo uses TailwindCSS to configure itself. Breakpoints, colors, font sizes, shadows and etc. configuration is provided by TailwindCSS config file. While all of the TailwindCSS utilities are available as classes, it's also useful to have some of them as SCSS variables and mixins:

Icon size helpers

Icon size helpers helps quickly changing the icon size on mat-icon elements. These helpers support both icon fonts and svg icons.

Class

  • .icon-size-[SIZE]

    By default, SIZE parameters are but not limited to: 12 14 16 18 20 24 32 40 48 56 64 72 80 88 96.

Mixin

To change the icon size from the SCSS files, you can also use the @include treo-icon-size($size) mixin. It has the same $size parameter rule as the helper class.

In order to use this mixin, you need to target the .mat-icon element and you must turn off the View encapsulation of your component.

Elevation helpers

These helpers can be used to quickly add an elevation to the elements.

Mixin

Use the @include treo-elevation($elevation, $important, $color) mixin to add an elevation based on box-shadow.

Parameters

  • $elevation

    Default elevation levels from TailwindCSS

  • $important

    Whether to add !important tag to the generated rule.

  • $color

    Color of the shadow, can be any valid SCSS color.
