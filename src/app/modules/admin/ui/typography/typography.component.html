UI ELEMENTS

Typography

When building a web application, you don't want to have the default HTML element styles interfering with your design. This is mainly because those style are designed for building web sites therefore Treo provides a typographic reset and some limited styling by default.

This way, you can build complex layouts and application interfaces without needing to reset element styles.

Body

The default body text is set to 14px. This allows building more complex and dense layouts without needing too much screen space. Applications usually need to show a lot more information on one page compared to, let's say, a blog post. This requires more denser layouts hence the smaller default body font size.

Headings

These headings are specifically designed for application and page titles. They are a lot bigger and bolder compared to content headings.

Links

Since links can also be used as buttons, <a> tag has been reset to not have a color nor an underline by default. If you wish to actually show a link, use .link class.

Rich text

As we mentioned before, by default, Treo resets all typographic styles. Because of this, a rich-text content with paragraphs, lists, links, titles and other HTML elements will look bad and they won't be properly styled.

To overcome this issue, there is a helper class called .rich-text. Anything put into a container with this class will be treated as rich-text content and will be styled accordingly.
